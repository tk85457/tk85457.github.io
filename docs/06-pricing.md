# Chapter 06 — Pricing & Credit System

## Objective
Design a transparent credit-pack purchasing experience that is easy to understand, trustworthy, and optimized for conversion.

## Pricing Philosophy
- Sell credits instead of subscriptions.
- Clearly explain what credits represent.
- Avoid hidden fees.
- Show approximate watch time for each pack.

## User Flow
Pricing Page → Select Credit Pack → Secure Checkout → Credits Added → Dashboard Updated.

## Credit Packs
Each pack should display:
- Credits included
- Estimated translation time
- Price
- Best value badge (when applicable)

## Calculator
Interactive slider converting credits into approximate usage time.

## Trust
- Secure payment provider
- Refund policy
- FAQ
- Purchase history available in dashboard.

## Success
Users should understand pricing without reading lengthy explanations.
