# SyncDub AI Product Bible

# Chapter 10 — Motion System

## Objective
Create a unified motion language that communicates state, improves usability, and reinforces the SyncDub AI brand.

## Principles
- Motion has purpose.
- Fast, smooth, and subtle.
- Never distract from content.
- Respect reduced-motion preferences.

## Motion Categories
- Page transitions
- Hover states
- Button feedback
- Modal transitions
- Toast notifications
- Loading states
- Waveform animations
- Connection status

## Timing
- Hover: 120–180ms
- Standard transition: 200–300ms
- Page transition: 250–400ms

## Easing
Use consistent easing curves across the product.

## Performance
Prefer GPU-accelerated transforms and opacity animations. Avoid layout-thrashing animations.

## Outcome
Users should perceive the interface as responsive, polished, and predictable.
