# SyncDub AI Product Bible

# Chapter 03 — Information Architecture

## Objective

Define the complete structure of the SyncDub AI ecosystem so every page, flow, and interaction has a clear purpose.

## Primary User Journey

Visitor → Homepage → Interactive Demo → Install Extension → Sign In → Free Credits → Translate First Video → Purchase Credits → Returning Customer.

## Website Structure

- Home
- Features
- Demo
- Pricing
- Languages
- Platforms
- Download
- Dashboard
- Credits
- Billing
- History
- Settings
- Documentation
- FAQ
- Blog
- Contact
- Privacy
- Terms

## Navigation Principles

- Keep the primary navigation focused.
- Surface the installation CTA on every marketing page.
- Minimize clicks to start using the product.
- Separate marketing, product, and support content.

## Dashboard Sections

Overview, Credits, Usage, Purchase History, Translation History, Settings, Account, Billing, Support.

## Documentation

Organize documentation by user tasks rather than internal implementation details.

## Future Expansion

Reserve architecture for mobile apps, desktop apps, public API, enterprise administration, and creator tools without restructuring the navigation.
